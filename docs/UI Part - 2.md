\## 3. 🏗 Архитектура и стек технологий



* Backend: PHP 8.x, PDO, MySQL 5.7+
* Frontend: HTML5, CSS3, Bootstrap 5, Vanilla JS (fetch)
* Архитектура: MVC, Repository Pattern (DAL), разделение логики и представления
* Инфраструктура: Git, хостинг Beget, phpMyAdmin
* Безопасность: CSRF-токены, подготовленные выражения, серверная валидация, экранирование вывода



\## 4. 🔑 Ключевые реализации

\### 4.1. Расчёт доступных слотов



Метод AppointmentRepository::getAvailableSlots() генерирует сетку времени с 09:00 до 18:00 (шаг 30 мин), исключает обед (13:00–14:00) и вычитает уже занятые интервалы конкретного специалиста.



\### 4.2. Конкурентная защита при создании записи



```php

$this->pdo->beginTransaction();

try {

&nbsp;   $check = $this->pdo->prepare("SELECT COUNT(\*) FROM appointments WHERE specialist\_id = ? AND appointment\_datetime = ? AND status != 'отменено'");

&nbsp;   $check->execute(\[$data\['specialist\_id'], $data\['appointment\_datetime']]);

&nbsp;   if ($check->fetchColumn() > 0) throw new \\Exception("Время только что занято!");

&nbsp;   // INSERT ...

&nbsp;   $this->pdo->commit();

} catch (\\Exception $e) {

&nbsp;   $this->pdo->rollBack();

&nbsp;   throw $e;

}

```



\### 4.3. Перенос записи (reschedule)



Метод проверяет доступность нового слота, обновляет appointment\_datetime, освобождает старое время и записывает событие в лог (если таблица appointment\_log существует).



\### 4.4. Экспорт отчётов в CSV



Заголовки Content-Type и Content-Disposition отправляются до любого вывода HTML. Файл формируется через fputcsv() с добавлением UTF-8 BOM для корректного открытия в Excel.
